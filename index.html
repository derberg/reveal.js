SSG vs DITA

DITA is an XML based standard

SSGs are tools that generate output basing on a given input

SSG and DITA comparison
Content Reuse

  • DITA: conref 
    										
conref="../reusables/topics/reusables-importing-data.dita#reusables-importing-data/smart-paste-para"
  • SSG: includes (jekyll) or partials (docpad and hugo) 
    										
<%- @partial('feedback_url') %>

Metadata

  • DITA: used for indexing
  • SSG: managing and identifying a document
    In DocPad doesn't have to be explicitly set in file 
    										
---
title: My blog post about cats
tags:
	- animal
---
your content written in **Makrdown** or HTML

Maps - relationship tables

  • DITA: ditamap and map element
  • SSG: docpad-plugin-related and tags metadata

Information Typing

  • DITA: default like glossary or task + anything through specialization
  • SSG: through metadata and folder structure 
    										
---
title: Creating an object
type: Tutorial
---

Specialization

  • DITA: possible
  • SSG: possible
